# React Best Practices

React 또는 Next.js 작업에서 성능에 영향을 주는 변경을 판단할 때 이 스킬을 사용한다.

## 적용 절차

1. 루트와 작업 대상 폴더의 `AGENTS.md`를 먼저 확인한다.
2. 요청 범위와 관련된 규칙만 `AGENTS.md`에서 찾아 읽는다.
3. 비동기 워터폴, 번들 크기, 데이터 패칭, 리렌더링, JavaScript 연산 비용 순으로 검토한다.
4. 성능 규칙을 기계적으로 적용하지 말고 현재 React·Next.js 버전과 코드 동작을 확인한다.
5. 기존 동작, 공개 API, 타입 안정성, 주석과 프로젝트 스타일을 유지한다.
6. 변경 후 관련 lint, 타입 검사, 테스트 또는 빌드를 실행한다.

## 참조 문서

- 전체 규칙과 예제가 필요하면 `AGENTS.md`를 읽는다.
- 큰 문서이므로 먼저 제목이나 규칙 키워드로 검색하고 관련 구간만 읽는다.
- 프로젝트의 `AGENTS.md`와 충돌하면 프로젝트 지침을 우선한다.
